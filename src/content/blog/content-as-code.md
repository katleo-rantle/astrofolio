# Content as Code: Why My Blog Lives in a Separate Repo

Treating content as code has been one of the best decisions for my personal site.

## The Old Way vs New Way

Previously, every small blog post meant rebuilding and redeploying my entire site.

Now I can:

- Write posts in my main knowledge base (PKM)
- Push changes to a private repo
- Automatically sync them to my portfolio

## Benefits I’ve Seen

- Better separation of concerns
- Easier version control of my writing
- Cleaner commit history in my portfolio repo

This setup feels much more scalable.
